Display inline vs. inline-block

There are two types of elements in HTML, block elements and inline elements. Block elements, like div and p, can contain other elements, and take up as much horizontal space as they can. In other words, a block element tries to be as wide as it can on the page. Block elements also don't like to be next to each other but stack on top of each other on the page. You can think of them as having a carriage return at the end.

Inline elements, on the other hand, can be inside other elements, but can't contain block elements. Examples of inline elements include span. Inline elements have a height, but try to be as small as possible in terms of width, and they don't mind having other elements next to them because they don't have that "carriage return" at the end.

The display property in css can force block elements to behave more like inline elements in terms of how they display on the page, or make inline elements act like blocks. Display defines the type of element box an element generates in the layout. There are lots of different possible values for display, but here we're interested in the difference between the values inline and inline-block.

To understand the difference it's helpful to think about the differences between how blocks and inline elements display. Blocks take up as much room as they can and stack on top of each other. Inline elements minimize their size and display next to each other. An element given the display value of inline-block is sort of a hybrid. It will take up space, but allow other elements to sit next to it, in other words it will display inline.
